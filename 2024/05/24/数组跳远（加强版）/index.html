数组跳远（加强版）

发表于2024-05-24|更新于2024-05-24
|阅读量:

n你知道的，动态规划这个专题里面的题目不一定和动态规划有关。那为什么要把这个题目放在这个专题里面呢？是因为yg老师说这是动态规划的题目，虽然我看不出来这个题目跟动态规划有什么关系。

问题描述】

​ 对于一个具有 *n* 个元素的数组 *a* , 执行以下操作：

  • 首先，选择下标 i** （1 ≤ *i* ≤*n*）—— 设置为数组的开始位置，放一个标记在 i* 处（值 为ai***）
  • 当 *i* ≤ *n* 时，你的得分将增加 ***ai ，***且将标记向右移动 *ai* 个位置即下个位置是 *i* +*ai* ，继续这个过程直到 i>n
  • 如果 *i* >***n，***则结束操作

​ 例如， 如果 n* = 5且a*=[7, 3, 1, 2, 3]，则可以进行以下操作

  • 选择 i* = 1（**a1***为7），操作过程为，最后得分为 *a1* = 7
  • 选择 i* = 2（**a2***为3），操作过程为, 最后得分为 *a2* + *a5* = 6
  • 选择 *i* = 3，操作过程为, 最后得分为 *a3* + *a4* = 3
  • 选择 *i* = 4，操作过程为, 最后得分为 *a4* = 2
  • 选择 *i* = 5，操作过程为, 最后得分为 *a5* = 3

​ 请选择合适的开始位置，使得经过上述操作后可获得最大的分数。

【输入形式】

​ 输入的第一行为一个整数 *t* (1≤ *t* ≤ 104)，表示测试用例的组数。

​ 每个测试用例的第一行为一个整数 *n* (1≤ *n* ≤ 2×105)，表示数组 *a* 的元素个数

​ 接下来一行包含 n** 个整数 *a1*、*a2**、…、an* (1 ≤ ***a****i* ≤ 109)，表示数组 *a* 的元素

【输出形式】

对于每个测试用例，输出独立一行，表示选择合适的开始位置后经过上述操作可以获得的最大分数。

题目大概就是这么个事儿，这个加强主要体现在测试数据部分，可以看到数据规模很大，如果直接暴力解题，或许很难全部通过。其实我一开始就知道暴力求解无法通过，但还是义无反顾地A了上去。你知道的，人就是这样，当然这样做也没错，毕竟很多事试试才知道，万一就成了呢?人不能失去尝试的勇气，就像只狼不能失去忍义手。

但是现实很残酷，只过了6组测试数据。于是我开始思考，怎么优化呢？首先想到的是提升I/O流输入的性能，于是我添加了ios::sync_with_stdio(false);cin.tie(0);cout.tie(0);这一行代码，取消C++的I/O库与C的I/O库（即stdio.h）的同步，将cin和cout解绑。然而并没有什么卵用，还是只能通过6组测试数据。
